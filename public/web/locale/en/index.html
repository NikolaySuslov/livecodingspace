Virtual Worlds

To begin collaborative coding in virtual space, just start one of the listed prototypes and connect to it from another browser window using the generated link. The link will apper near the Start button. By default a tiny Reflector server is used for creating a Virtual World instances. Check Settings for the correct Reflector server adress (https://time.livecoding.space by default).

NEW! You could try out to connect by using the new experimental totaly decentralized solution provided by Krestianstvo Luminary. That will allow to create Virtual World Instances just within Web Browser by a client, using any avaliable decentralized storage GunDB on internet (https://db.livecoding.space by default). No any Reflector server needed . In the near future it will be extended with support of securing world instances lists, clients lists, message streams with P2P identities. Turn on the Switch for using Krestianstvo Luminary. If a connection to the Virtual World instance is not estableshed from the first time, try to reload the page in the web browser again.
